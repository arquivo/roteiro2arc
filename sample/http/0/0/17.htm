Participação no serviço X.500 da RCCN

Perfil de Participação

As organizações (académicas ou não) podem participar no projecto piloto de várias maneiras:
  • Instalação de DSA/DUA próprio
  • Instalação dum DUA próprio e colocação da informação num dos DSAs de backbone da RCCN
  • Utilização de DUAs de acesso público e colocação da informação num DSA de backbone da RCCN
  • Utilização de DUAs de acesso público

DUA de acesso público

Existe presentemente um DUA de acesso público. O acesso é conseguido

  • Por telnet, para o host dsa.fccn.pt (endereço IP 193.136.5.1)
  • Por pad, para o endereço 02301031425 (TELEPAC) ou 20436840001025 (EMPB).
Utilizando o login dua acede-se a um menu onde é dada a opção de escolha entre diversos DUAs.

DUA próprio

Para instalar um DUA próprio é necessário que a organização tenha conectividade IP nacional num MacIntosh com MacTCP, num PC MS-DOS com o software FTP ou ainda uma máquina Unix.

É também possível manter um DUA próprio possuindo os requisitos hardware e software para instalar o ISODE (ISO Development Environment) tendo conectividade IP nacional ou acesso síncrono à Rede Pública de Dados (TELEPAC).

Colocação de informação em DSAs de backbone

Para colocar informação num DSA de backbone é necessário que exista na organização um gestor de dados que se responsabilize pela sua manutenção.

Procedimento a realizar

  • Contacto com o gestor do DSA Master PT para criação da organização e da entrada para o gestor de dados
  • Colocação, usando o DUA DM de acesso público da informação recolhida na respectiva organização
  • Manutenção da informação utilizando o DM
Os DSAs de backbone só mantêm informação sobre organizações com um número limitado de entradas (menos de 100). As organizações que ultrapassem esse limite devem instalar o seu próprio DSA.

DUA/DSA próprios

Para que uma organização possa manter a sua informação num DSA próprio é necessário:
  • Dispor de conectividade IP nacional ou acesso síncrono à Telepac
  • Dispor duma máquina Unix que suporte o ISODE 8.0
  • Instalar o ISODE 8.0 e o QUIPU
  • Contactar o gestor do DSA master PT para registo da sua Organização e DSA e,
  • Ligar-se à Directoria Global.
